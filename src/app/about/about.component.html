What is Clipshare?

Clipshare is a lightweight, modern video-sharing platform built as my capstone project. It allows users to upload videos, browse content, view details, and interact with a streamlined, responsive interface. I designed Clipshare to mimic the essential experience of larger video platforms but with a simpler, more focused approach to usability.

  • Fast, responsive UI built with Angular + Tailwind
  • Clean, modern layout designed for clarity and ease of use
  • Smooth video handling with Firebase storage
  • Dynamic routing and transitions for a native-app feel
  • Minimal friction when navigating, watching, or interacting with content

Deployment & Hosting

This project is deployed using Vercel as they allow free unlimited hosting for passion projects, perfect for presenting side projects like this.

Frontend (Client) Deployment

Clipshare is built as a near fully client side Angular application styled with Tailwind CSS. The production build is generated using Angular’s build pipeline, producing a highly efficient static bundle. Vercel serves this bundle through its global network, ensuring fast load times, automatic caching, and instant rollback capabilities. Client side routing is configured to rewrite all non file paths to index.html, allowing deep links and browser refreshes to work correctly.

Media Storage & Streaming

All video uploads and related metadata are stored in Firebase. Firebase Storage is responsible for handling large media assets, managing upload sessions, delivering optimized streaming URLs for playback, as well as user authentication. Videos are distributed through Firebase’s global CDN, ensuring smooth and consistent performance across devices. This approach keeps backend complexity low while providing a reliable and scalable media layer for the application. A bonus is its also free.

CI/CD & Workflow

The project is managed through GitHub, and every push triggers an automated deployment pipeline in Vercel. Feature branches generate preview deployments that allow testing changes in an environment before merging. When updates are merged into the main branch, Vercel automatically builds and publishes a fresh production version. This workflow enabled rapid iteration, safe testing, and continuous delivery throughout development.
